Human vision is “trichromatic”

  • WHY RGB？
  • human eye has three classes of colour photoreceptors, called → cones
    • 3 diff types of cones have diff sensitivities

These sensitivities are important → camera sensors are – in a certain technical sense - designed to mimic the sensitivities of cones

  • transmission spectrum: the fraction of incoming light that is transmitted
  • absorption: talk about this more in this class → fraction of light that arrives at a surface OR volume and is absorbed (as a function of wavelength)

Analog-to-Digital conversion

this signal (including noise) is then digitized, namely quantized and then coded in binary

  • use more bits for each RGB value for better resolution photos

STEP 2: White balancing

we want whites object to appear white

Example.

if you tell the camera that the light source is a tungsten light bulb (which emits reddish light), then the camera would multiply all the red values by some number that is less than 1, to compensate for the reddish light

  • correct the amount of white lights, so it looks more like normal
    • example.
      • set to tungsen: compensate by multiplying the R channel by a smaller number to cancel the light source effect, which is assumed to be dominated by long wavelengths (red). This makes the image appear very blue.

STEP 3: Map to Standard Color Space: sRGB

transform to a standardized color space → a common one is sRGB

2 STEPs:

  1. reweighting each RGB value to some (arbitrary) standard
  1. non-linear mapping (”gamma”) is applied to the RGB values

STEP 3.2: Display Non-linearity (Gamma) → Gamma Correction

gamma exponent is typically about 2

  • When each RGB value is sent to a display, the intensity is proportional to the RGB values raised to some power (called the “gamma” of the display)

Q: Why gamma correction?

A: RGB display doesn’t do continuous intensities, but ONLY 256 possible ones

→ gamma allows you to represent the lower intensity values with more precision than the upper intensity values
