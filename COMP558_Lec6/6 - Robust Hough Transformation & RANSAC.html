2. Hough Transform (1962)

for each pt, vote for all the lines that pass near the point
  • define a discrete set of candiates lines (r,θ)(r,\theta)(r,θ)﻿

  • for each sample point (xi,yi)x_i, y_i)xi​,yi​)﻿
    • for each line near that point, parameterized (r,θ)(r,\theta)(r,θ)﻿ cast one vote for this line
  • return the line (r,θ)(r,\theta)(r,θ)﻿ that has MAXIMUM COUNT

We could penalize points by their squared distance from the line (or their absolute distance) up to some distance, say dmaxd_{max}dmax​﻿

3. RANSAC (Random Sample Consensus)

by Fischler and Bolles

RANSAC idea:

randomly pick 2 diff points & fit a line to the 2 pts

Q: then…. When can say that we have a good line model?

A: good if many other points lie near

ex 1. fitting a line thru 2 points

Q: Why use the minimal number of points to fit?

A: TRADEOFF!!!

  • use more points → potentially better model
    • BUT onlly if extra points are inliers
  • use more points → greater chance to be outlier → makes poor fit

RANSAC Algorithm: ✅

Application of RANSAC: Vanishing Point Detection

EASY: pick 2 random edges get the intersection, count # edges passes nearby from the rest N−2N-2N−2﻿ edges

  • good if most edges nearby

RANSAC is a probabilistic model

  • let www﻿ be fraction of pts that ARE INLIERS.

    (We assume that there is one correct model and “points” are either inliers or outliers.)

  • If randomly sample nnn﻿ points to fit a model → then p(p(p(﻿all n samples are inliers) =wn= w^n=wn﻿
  • so p(p(p(﻿at least 1 of n is outlier) = 1−wn1-w^n1−wn﻿
